# Feature Ideas

모의 투자 시뮬레이션 프로젝트 확장 아이디어 모음.

---

## 1. 성과 분석 고도화

현재는 수익률과 순위만 표시. **투자 퀄리티** 지표를 추가하면 전략 간 비교가 훨씬 깊어진다.

### 추가 지표
- **샤프 비율 (Sharpe Ratio)**: 위험 대비 수익. "A가 수익률은 높지만 리스크도 컸다" 같은 인사이트
- **최대 낙폭 (Max Drawdown)**: 고점 대비 최대 하락폭. 전략의 방어력 측정
- **승률 (Win Rate)**: 수익 거래 / 전체 거래. 전략별 적중률
- **알파 / 베타**: E(벤치마크) 대비 초과수익(알파)과 시장 민감도(베타)
- **변동성 (Volatility)**: 일간 수익률의 표준편차

### 시각화
- 투자자별 지표를 **레이더 차트**로 한눈에 비교
- 대시보드 요약 카드에 샤프비율, 최대낙폭 추가

### 구현 난이도: 중
- `daily_reports`에 이미 일별 총자산 데이터가 쌓이고 있어 계산 가능
- 데이터가 최소 2~3주 이상 쌓여야 의미 있는 수치

## 2. 투자자 간 상관관계 & 대결 구도

### 포지션 분석
- **포지션 겹침률**: "A와 D는 정말 반대로 가고 있나?" 실제 데이터로 검증
- **종목 인기도**: 각 종목을 보유 중인 투자자 수 시각화 (컨센서스 지표)
- **수익률 상관관계 매트릭스**: 투자자끼리 얼마나 같이 움직이는지 히트맵

### 대결 & 게이미피케이션
- **1:1 대결 페이지**: "A vs D", "F vs G" 같은 직관적 비교. 누적 수익 곡선 오버레이
- **주간 MVP / 꼴찌**: 매주 가장 잘한/못한 투자자 하이라이트
- **연승 기록**: "A가 3일 연속 1위!", "D가 5일 연속 수익!" 같은 스트릭 추적
- **마일스톤 뱃지**: "첫 수익 달성", "총자산 600만원 돌파", "첫 손절" 등 이벤트 트래킹

### 구현 난이도: 중
- 히트맵/상관관계는 recharts 또는 별도 라이브러리 필요
- 대결 페이지는 새 라우트 추가 (`/versus/[a]-vs-[b]`)

## 3. 섹터 & 종목 분석 페이지

### 섹터 뷰
- **섹터 히트맵**: 반도체/바이오/방산 등 섹터별 일간 등락을 색깔 타일로 시각화
- **섹터별 투자 비중**: 각 투자자가 어느 섹터에 얼마나 배팅하고 있는지
- **섹터 수익 기여도**: 어떤 섹터가 전체 수익에 가장 크게 기여했는지

### 종목 뷰
- **종목 상세 페이지** (`/stocks/[ticker]`): 가격 차트 + 보유 투자자 목록 + 거래 내역
- **종목별 수익 기여도**: 각 투자자에게 이 종목이 얼마나 기여/손실을 줬는지

### 구현 난이도: 중~높
- stock_universe에 이미 sector 정보 있음
- 종목 상세 페이지는 새 라우트 + yfinance 히스토리 활용

## 4. 시간축 확장 (리포트 & 캘린더)

### 주간/월간 자동 리포트 ✅ 주간 구현 완료 (`scripts/weekly_report.py`)
- 한 주/한 달 동안의 성과를 AI가 자동 요약
- "이번 주 A는 모멘텀 전략이 통했고, D의 역발상은 초반 손실 후 후반 반등..."
- Supabase에 `weekly_reports` / `monthly_reports` 테이블 추가

### 달력 히트맵
- GitHub 잔디밭 스타일로 일별 수익률을 색깔로 표시
- 빨강 = 수익일, 파랑 = 손실일, 진할수록 큰 변동
- 투자자별 또는 전체 합산 뷰

### 구현 난이도: 중
- daily_reports 데이터 활용
- 캘린더 히트맵은 커스텀 컴포넌트 또는 라이브러리 사용

## 5. 새로운 투자자 추가

전략 다양성을 높이기 위한 후보 투자자들.

### H 박기술 - 기술적 분석 (Technical Analysis)
- RSI, MACD, 볼린저밴드 등 기술적 지표 기반
- 리밸런싱: 매일
- "차트쟁이 vs AI 판단" 대결 구도
- 구현: yfinance 히스토리 데이터로 지표 계산 필요

### I 최배당 - 배당 투자
- 배당수익률 상위 종목 위주
- 리밸런싱: 분기 (90일)
- 배당주의 장기 복리 효과 검증
- 참고: stock_universe에 배당 데이터 추가 필요

### J 한따라 - 스마트머니 추종
- 외국인/기관 순매수 상위 종목 추종
- 리밸런싱: 매주
- "스마트머니를 따라가면 이길 수 있나?" 실험
- 참고: 외국인/기관 매매 데이터 소스 필요 (네이버 금융 등)

### 구현 난이도: 중~높
- 투자자 추가 자체는 profiles 테이블에 레코드 추가로 간단
- H는 기술 지표 계산 로직, I는 배당 데이터, J는 외부 데이터 소스 필요

## 6. 인터랙티브 기능

### "만약에" 시뮬레이터
- 사용자가 직접 20종목 비중을 정해서 가상의 8번째 투자자로 참여
- 실제 투자자들과 성과 비교
- 웹에서 슬라이더로 비중 조절 → 백테스트 결과 표시

### 투표/예측 게임
- "내일 1등은 누구?" 방문자 투표
- 다음날 결과와 비교하여 적중률 표시
- 간단한 참여형 콘텐츠

### 구현 난이도: 높
- 사용자 입력 처리 + API 라우트 필요
- 투표는 별도 인증/세션 관리 고려

## 7. 스토리텔링 & 콘텐츠 자동 생성

### AI 데일리 코멘터리
- 매일 시뮬레이션 결과를 바탕으로 짧은 마켓 코멘터리 자동 생성
- "오늘 D 장반대가 역발상으로 2.3% 수익을 올리며 1위에 올랐다. 반면 A 강돌진은..."
- 대시보드에 "오늘의 요약" 섹션으로 표시
- Supabase에 `daily_commentary` 테이블 추가

### 투자자 일기
- 각 투자자 캐릭터가 1인칭으로 쓰는 일기 형식의 AI 생성 콘텐츠
- "오늘 SK하이닉스를 추가 매수했다. 반도체 슈퍼사이클이 온다고 확신한다." (A 강돌진)
- 투자자 상세 페이지에 "일기" 탭 추가

### 주간 뉴스레터
- 주간 성과 + AI 코멘터리를 이메일 뉴스레터로 자동 발행
- 기존 `send_email.py` 확장

### 구현 난이도: 중
- 시뮬레이션 파이프라인에 코멘터리 생성 스텝 추가
- Claude API로 자연어 생성 후 Supabase 저장

## 8. 알림 & 구독 채널 확장

### 텔레그램 봇 ✅ 일간 알림 구현 완료 (`scripts/send_telegram.py`)
- 일간 결과 요약 자동 발송
- `/status` 명령어로 현재 순위 조회
- `/investor A` 명령어로 특정 투자자 현황 조회

### 카카오톡 알림
- 카카오 알림톡 or 친구톡으로 일간 요약 발송

### RSS / 블로그
- AI 코멘터리를 블로그 형태로 공개
- RSS 피드 제공

### 구현 난이도: 중~높
- 텔레그램 봇은 python-telegram-bot 라이브러리로 비교적 쉬움
- 카카오톡은 비즈니스 채널 등록 필요

## 우선순위 제안

### Phase 1 (데이터 축적 기간, 1~2주)
1. **AI 데일리 코멘터리** - 매일 콘텐츠가 쌓여 가치 증가
2. **달력 히트맵** - 시각적 임팩트 크고 구현 간단

### Phase 2 (데이터 2주 이상)
3. **성과 지표 고도화** - 샤프비율, 최대낙폭 등 의미 있는 분석
4. **1:1 대결 & 주간 MVP** - 게이미피케이션

### Phase 3 (안정화 후)
5. **섹터 히트맵 & 종목 분석 페이지** - 대시보드 확장
6. **새로운 투자자 추가** - H/I/J 전략 다양성
7. **텔레그램 봇** - 외부 채널 확장

### Phase 4 (장기)
8. **"만약에" 시뮬레이터** - 인터랙티브 참여
9. **주간 뉴스레터** - 콘텐츠 마케팅
